The Big Picture

Why we built an AI content engine

Agility's content needs outpaced what one person could do manually. Weekly blog posts, 7 social posts per week, monthly newsletters, case studies on demand, engagement monitoring, HubSpot nurture emails — all needing to stay on-brand, data-driven, and consistent. Every week. Without fail.

The answer wasn't hiring a content team. It was building a system — an AI-powered orchestration layer that handles the research, writing, quality checks, formatting, and publishing. The strategic decisions (what to write about, how to position it, what angle to take) stay human. Everything else is automated.

Key insight

AI is not "write me a blog post." It's an orchestration layer. Research feeds writing, writing feeds QC, QC feeds publishing, publishing feeds analytics, analytics feeds the next cycle. Each step is automated and connected.

For Sales

This means the content you're sharing with prospects is always fresh, keyword-optimized, and published on a consistent cadence. No stale collateral. The blog targets keywords your prospects are actively searching, and every piece goes through brand-safety QC before it goes live.

For Strategy

Client-facing implications: we can produce case studies, thought leadership, and client-specific content at scale. You can request content via the Hub, and the pipeline handles research, writing, and formatting automatically.

For Marketing

This is the system you'll interact with most. Understanding the pipeline means you can request briefs, influence topics through keyword strategy, and leverage the content library. Every pipeline feeds the next one — blog content becomes social posts, social themes become HubSpot emails.

The Philosophy

How to think about AI-powered content

Three principles drive everything in the system. Understanding these will change how you think about using AI — not just for content, but for any work.

Principle 1: The De-Sloppify Pattern

The biggest mistake people make with AI is front-loading prompts with what not to do. "Don't be verbose. Don't use jargon. Don't mention competitors." This actually degrades output quality — the model hedges and second-guesses itself.

The common approach

"Write a blog post about CTV measurement. Don't be verbose. Don't use jargon. Don't mention competitors. Don't use em dashes. Don't start with 'In today's...' Avoid cliches. Don't be too salesy..."

Result: Hedging, generic, lifeless content that reads like it was written by committee.
The de-sloppify approach

Pass 1 — Generate: "Write a confident, data-driven article about CTV measurement for enterprise CMOs. Active voice, specific examples, trust the reader."

Pass 2 — Clean up: Automated QC catches banned words, removes competitor mentions, tightens verbose passages, enforces voice rules.

Result: Natural, authoritative content that passes brand safety checks.
Why this matters for you

This applies to any AI work, not just content pipelines. When you're using Claude for research, writing, or analysis — tell it what you want, not what to avoid. Then review the output and iterate. Two clean passes beat one over-constrained attempt every time.

Principle 2: Two-Layer Quality Control

Quality isn't one check — it's two distinct layers that serve different purposes.

Layer 1
Content Safety

String-matching and list-based checks that enforce absolute rules.

  • Banned words and phrases (corporate jargon, cliches, hedging language)
  • Competitor name and URL filtering (The Trade Desk, StackAdapt, Criteo, etc.)
  • Promotional language detection in engagement content
  • Banned openers ("In today's landscape..." "Here's the thing...")
Layer 2
Structural QC

Pattern-based checks that catch subtle quality issues.

  • Negation-pivot patterns ("Not X, but Y" — sounds defensive)
  • Missing calls to action
  • Stat and claim accuracy (only pre-approved statistics can be cited)
  • Link integrity and URL validation

Principle 3: Automate the Repetitive, Preserve the Strategic

The system has a clear boundary between what's automated and what stays human.

The system handles
  • Keyword research and gap analysis
  • Topic research with current data
  • Drafting and formatting
  • Quality checks and safety filters
  • Document creation and publishing
  • Cross-platform distribution
  • Scheduling and notifications
Humans decide
  • Which topics matter this quarter
  • How to position against competitors
  • What angle resonates with each audience
  • Whether a case study is ready to share
  • Final editorial review before publish
  • Strategic pivots based on market shifts
  • Client-specific messaging and tone

The Knowledge System

The pipelines don't start from zero each time. A structured knowledge vault stores strategic context — content strategy, audience personas, competitive positioning, channel playbooks, and accumulated learnings from past performance. When Claude writes content, it reads the relevant strategy files first, so every piece is informed by what we know works.

This is the compounding advantage: every feedback cycle (what performed well, what got edited, what clients responded to) gets captured and feeds the next generation of content. The system gets smarter over time.

For Sales

Think of this like having a research team that never sleeps. Content is data-backed because the system reads keyword gaps, competitor positioning, and performance data before writing anything. When you share a blog post with a prospect, it was written with their search behavior in mind.

For Strategy

The knowledge system stores client intelligence, meeting insights, and industry trends. When the system writes about a topic, it already knows our positioning and what's worked before. You can contribute to this by flagging client wins and market observations.

For Marketing

Understanding the de-sloppify pattern and QC layers helps you write better prompts for any tool, not just our pipelines. The two-pass approach (generate freely, then clean up) is the single most impactful technique for getting high-quality AI output.

What This Means for You

Role-specific takeaways and actions

As someone on the sales team, the automation suite is your content supply chain. Here's how to leverage it.

1

Use the Content Library

Every piece of content gets AI-tagged by industry, use case, and business size. Before a prospect meeting, search the library for relevant case studies, blog posts, and one-pagers. Filter by vertical (Retail, QSR, Automotive, Healthcare, etc.) or pain point (Brand Lift, CTV/OTT, Incrementality Testing, etc.).

2

Share blog posts that match prospect intent

The blog pipeline targets keywords your prospects are actively searching. When you see a blog about "CTV measurement" or "incrementality testing," that topic was chosen because SimilarWeb data showed prospects searching for it. Share these — they're designed to address real questions.

3

Request case studies for specific verticals

Have a client win? Submit the data through the Hub intake form. The pipeline generates a structured case study — headline stats, challenge, approach, results, timeline — formatted and ready for the sales deck. No waiting on the marketing queue.

4

Trust that content is current

Everything on agilityads.com is published through Framer CMS and is always live. The blog publishes weekly, social goes out daily, and the newsletter drops monthly. When you link a prospect to our content, it's fresh — not a PDF from six months ago.

As a Strategy Consultant, the automation suite gives you content leverage for client relationships and thought leadership.

1

Submit case study intake forms

When a campaign delivers strong results, submit the data through the Hub. Include the challenge, approach, key metrics, and timeline. The pipeline handles the writing, formatting, and QC — you'll get a polished case study back that's ready for client presentations.

2

Leverage thought leadership for client relationships

The newsletter and blog pipeline produce consistent thought leadership that you can share with clients. When a blog post covers a topic relevant to a client's challenge, forward it. It positions Agility as a strategic partner, not just a vendor.

3

Contribute to the knowledge system

The knowledge vault stores client intelligence and market observations. When you learn something in a client meeting — a new competitive dynamic, a shifting budget priority, a successful approach — flag it. It feeds the next cycle of content and makes the system smarter.

4

Use engagement monitoring insights

The Reddit and Quora engagement pipelines surface conversations happening in your clients' industries right now. These threads show you what practitioners are asking about, what frustrations they have, and where opportunities exist for Agility's solutions.

As a marketing team member, you have the most direct interaction with the automation suite. Here's how to get the most from it.

1

Submit blog briefs via the Hub

Have a topic the blog should cover? Submit a brief through the Hub with the topic, target keyword, audience, angle, and any proof points. The pipeline picks it up and runs the full research-to-publish cycle, informed by your strategic direction.

2

Influence the content calendar

The keyword refresh runs monthly, but you can steer topic selection by updating the strategy files. If there's a new campaign theme, a product launch, or a market shift — flag it and the next content cycle adapts. The system is flexible, not rigid.

3

Review content in the Hub before it goes live

Everything — social posts, engagement comments, blog drafts — queues in the Hub for review before publishing. This is your editorial control point. You can approve, edit, or reject any piece. The automation handles volume; you control quality.

4

Close the feedback loop

The feedback analysis pipeline compares generated vs. final edited versions to identify patterns. If you consistently edit out a certain phrase or rewrite a certain section, the system learns. Your edits make the next batch of content better.

Claude Code vs. Claude Chat

Stop burning tokens — use the right tool

Right now, most of the team uses Claude in Chrome — either the chat interface or cowork mode. This works, but it's expensive and inefficient. You spend tokens re-explaining context that Claude Code would read automatically from your project files.

Claude Chat (Browser)
Good for quick questions, brainstorming, and conversations where you don't need files or project context.
Use when:
  • Quick question, no files involved
  • Brainstorming ideas or concepts
  • Analyzing text you paste in
  • One-off research questions
Claude Code (Terminal/Desktop)
Reads your project files automatically. Builds, edits, and runs things. Remembers context via CLAUDE.md. Dramatically more efficient for real work.
Use when:
  • Working with files, documents, or a project
  • Multi-step tasks (research + write + format)
  • Anything you'll do more than once
  • Building automations or workflows

Which tool should I use?

Does your task involve files, documents, or a project folder?

Will you need to do this task (or something similar) again?

Is this a quick question or an extended work session?

3 Rules for the Team

1

Start in Claude Code for any task that involves files or context

If you're working on documents, spreadsheets, presentations, or any project with multiple files, Claude Code reads the project folder automatically. No more copy-pasting context into chat.

2

Use CLAUDE.md to front-load context

Create a CLAUDE.md file in your project folder with your role, preferences, and rules. Claude reads it every time you start a session — no re-explaining. Type /init to auto-generate one from your project.

3

Use /compact when conversations get long

Long conversations eat tokens fast. Type /compact to compress the conversation history while keeping all context. This dramatically extends how much work you can do in a single session.

For Sales

Claude Code example: Need a custom pitch section for a QSR prospect? Open Claude Code in your templates folder — it reads your existing decks and generates a tailored section using our content library data. Chat example: Quick objection response or competitive positioning question? Chat is fine for that.

For Strategy

Claude Code example: Building a client strategy doc? Claude Code reads your project files, past presentations, and client intel to produce a comprehensive document. Chat example: Quick brainstorm on campaign positioning or measurement approach? Chat is fine.

For Marketing

Claude Code example: Editing content pipeline output, adjusting templates, or building a new automation? Always Claude Code — it understands the full project. Chat example: Ideating campaign concepts or analyzing a competitor's messaging? Chat works.
