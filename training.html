The Method

Context beats prompts. Every time.

A good prompt gets you a good answer. Good context gets you a good answer every time, without re-explaining anything. That's the difference between using Claude as a chatbot and using it as a system.

01
CLAUDE.md — your persistent instruction file

Claude Code reads a file called CLAUDE.md at the start of every session. It's a plain text file where you write who you are, what you're working on, and how Claude should behave. Claude follows these rules automatically, every session, without you saying a word.

This is the biggest productivity gain in Claude Code. Browser chat forgets everything when you close the tab. Claude Code remembers.

CLAUDE.md — example for a sales role
# My Role
Account Executive at Agility Ads. I sell to enterprise CMOs and VPs of Marketing with $5M+ ad budgets.

# What I Need Help With
- Prospect research briefs before meetings
- Personalized outreach emails (cold + follow-up)
- Competitive positioning against TTD, DV360, StackAdapt

# Rules
- Never mention competitor names positively
- Lead with data and specific results
- Write at a CMO reading level, not a marketer level
- Reference our case studies when relevant

# Key Stats I Use
- $6 ROI per $1 in brand advertising (Binet & Field)
- National Fitness: $11.2M incremental revenue
- 2.2x conversion lift vs placebo at 6 months
How to create yours

Open Claude Code in your project folder and type /init. Claude analyzes your files and generates a starter CLAUDE.md. Then open the file and add your role, rules, and key context. When Claude makes a mistake, tell it to add the correction to CLAUDE.md — the file improves over time.

02
Two passes, not one

Pass 1: Tell Claude what you want. Be specific about audience, format, and tone. Don't list what to avoid.

Pass 2: Review the output. Fix what's wrong. Or ask Claude to clean it up in a second pass.

Stuffing a prompt with "don't do X" makes the output hedge and second-guess. Negative instructions degrade quality. Generate freely, then clean up.

Weak
"Write a blog post. Don't be verbose. Don't use jargon. Don't mention competitors. Avoid cliches..."
Strong
"Write a 2,000-word article for enterprise CMOs about CTV measurement. Active voice. Lead with data." Then: "Review this and tighten any section over 3 paragraphs."

04
Power user commands
/init
Auto-generates a CLAUDE.md from your project. Run this first.
/compact
Compresses conversation history. Pro tip: add instructions like /compact Focus on the modified files and key decisions to control what survives compression.
Shift+Tab
Toggles Plan Mode. Claude plans but doesn't change files. Use before complex tasks — reduces trial-and-error token waste by 20-30%.
/clear
Resets conversation context. Use between unrelated tasks — mixing topics in one session degrades output on both.
/btw
Ask a side question that never enters conversation history. Zero context cost. Perfect for "wait, what does this term mean?" without polluting your main task.
Custom skills
Save repeatable workflows as /slash-commands. Example: a /prospect-brief skill that always runs your exact research + formatting sequence.
Memory
Claude remembers corrections across sessions. When it makes a mistake, tell it — it learns and won't repeat it next time.
claude --continue
Resume your last session with full context. Great for picking up where you left off after a break.
75% rule
Sessions that stay under 75% context produce higher-quality output than those that push to 100%. Run /compact at 50%, not when you're already maxed out.
05
Permission modes — how much freedom to give Claude

You control how much autonomy Claude has. Start cautious, increase as you build trust.

🙋
Ask first Default
Claude explains each action and waits for your approval. Best while you're learning.
📋
Plan first Shift+Tab
Claude shows the full plan. You approve once. It runs everything. Best for multi-step tasks.
🚀
Full auto Shift+Tab ×2
Claude works start to finish with no interruptions. Best when you trust the task completely.
Start with Ask First

Watch what Claude does for a few sessions. Reading its explanations builds your understanding of what's happening. Once you're comfortable, switch to Plan or Full Auto for speed.

Five things that hurt your output

1
The kitchen sink session

Mixing prospect research, email drafting, and report analysis in one conversation. Old context about Task A degrades work on Task B. Fix: /clear between unrelated tasks.

2
The correction spiral

Correcting Claude 3+ times on the same issue. Each failed attempt stays in context and confuses future attempts. Fix: after 2 corrections, /clear and write a better initial prompt.

3
Vague requests

"Make this better" costs 3 rounds of clarification. "Rewrite the intro to lead with the 47% CTV growth stat, keep it under 80 words" costs 1 round. Be specific about audience, format, length, and tone upfront.

4
Overstuffed CLAUDE.md

There's a ~150-200 instruction budget before compliance drops. If your CLAUDE.md is too long, important rules get lost. For each line, ask: "Would removing this cause mistakes?" If not, cut it.

5
Skipping Plan Mode

Jumping straight to execution on complex tasks causes 20-30% more token waste from trial-and-error. Press Shift+Tab first. Two minutes of planning saves twenty minutes of rework.

How to give Claude feedback

Vague feedback wastes rounds. Specific feedback gets fixed in one pass. Point to the problem. Say what you want instead.

Costs 1 round

Chat vs. Cowork vs. Code

Three tools. Different jobs. Use the right one.

Claude comes in three forms. Each one is built for a different type of work. Most of the team uses Chat for everything, which burns tokens fast and loses context between messages. Here's when to use each.

Chat
claude.ai in the browser
  • Quick question, no files
  • Brainstorming or ideation
  • Analyzing text you paste in
  • One-off research questions
Context resets every conversation
Cowork
Claude Desktop app
  • Multi-step knowledge work
  • Research across multiple sources
  • Document analysis and writing
  • When you want Code-level depth with a visual interface
Uses tokens faster than Code
Code
Terminal or Desktop app
  • Working with files or a project
  • Multi-step tasks with file context
  • Anything you'll do more than once
  • Building automations and tools
CLAUDE.md gives persistent memory
Why Code wins on value per token

Cowork and Code use similar amounts of tokens for similar tasks — both run the full agentic loop. The difference is what you get for those tokens. Code gives you CLAUDE.md (persistent context every session), /compact (compress history mid-session), and reusable automations (build once, run forever). Chat is cheapest per exchange but can't touch your files. Cowork is powerful for one-off deep work. Code is where repeatable value lives.

Quick decision: which tool?

Does your task involve files, documents, or a project folder?

Will you do this task (or something similar) again?

Quick question or extended work session?

Three rules

1

Files or context? Open Claude Code.

It reads your project folder. No copy-pasting. No re-explaining.

2

Create a CLAUDE.md file.

Your role, your preferences, your project rules. Claude reads it every session. Type /init to generate one automatically.

3

Type /compact when conversations get long.

Compresses history, keeps context. Extends your session by 3-5x.

What Claude Code can and can't do

Can
  • Read and write files on your computer
  • Run terminal commands
  • Search your entire project folder
  • Build automations and scripts
  • Call APIs (HubSpot, Slack, Google, etc.)
  • Remember preferences via CLAUDE.md
  • Generate .docx, .xlsx, .html, .csv, JSON
Can't
  • Browse the web in real-time
  • See your screen
  • Run on your phone
  • Access files outside your project folder
  • Guarantee perfect output every time
  • Remember anything without CLAUDE.md
Integrations (MCP servers)

Claude Code connects to your tools. With integrations called MCP servers, it reads Slack messages, creates HubSpot contacts, checks Google Calendar, pulls Ahrefs data, and more — all from the terminal. Ask your admin to set up the ones relevant to your role. This is how the Agility content engine connects to 10+ services from a single interface.
