Before we start: what is all of this?

Plain-English definitions for everything you're about to use
The Terminal
A text-based control panel

Instead of clicking icons, you type short commands. Think of it like texting your computer instead of tapping its screen. It's intimidating at first, but you only need a handful of commands to get started.

Claude Code
An AI assistant that lives in your terminal

You describe what you want in plain English — "add a contact form to my website" — and Claude reads your files, writes code, and makes it happen. It's like pair programming with an AI that already knows your project.

cmux
A smarter terminal app

cmux is the app you'll use to access your terminal. It's built for AI coding tools — with tab notifications that ping you when a task finishes, side-by-side panels, and a built-in browser, so you can walk away while Claude works.

Homebrew
An app store for terminal tools

Most developer tools aren't on the Mac App Store. Homebrew fills that gap — it's a free tool that lets you install things like cmux and Claude Code with a single command.

0

Install Homebrew

The app store for developer tools — you'll only do this once

Open your Mac's built-in Terminal app (press ⌘ + Space, type "Terminal", hit Enter). Paste this and press Enter:

Terminal
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It'll walk you through the rest. When it finishes, type brew --version and press Enter. If you see a version number, you're ready.

ℹ️
Already have Homebrew?

Skip ahead. To check, type brew --version and press Enter. If it shows a number, you have it.

1

Install cmux

Your new home base for Claude Code

cmux is a macOS terminal built from scratch for AI coding agents. The key feature: notification badges that alert you when a Claude task finishes — so you can hand something off and come back when it's done, instead of watching a progress bar.

Terminal
# Tell Homebrew where to find cmux
brew tap manaflow-ai/cmux
Terminal
# Install cmux
brew install --cask cmux

Once installed, open cmux from Spotlight (⌘ + Space → type "cmux"). From here on, use cmux instead of the built-in Terminal.

💡
Make cmux your default terminal

Go to System Settings → Desktop & Dock → Default terminal app and select cmux.

2

Install Claude Code

One command, ~30 seconds

Open cmux and run:

cmux
curl -fsSL https://claude.ai/install.sh | bash

When it finishes, confirm it worked:

cmux
claude --version

You should see something like claude 1.x.x. If so, move on.

What does that command do?

curl downloads a file from the internet. bash runs it. Together, it's like clicking "Download" then "Run" in a standard installer — just compressed into one step.

3

Log in

It's just like logging into any other app

Type claude and press Enter. The first time you run it, a browser window opens and asks you to sign in with your Anthropic account.

cmux
claude
💡
Browser didn't open?

Press the letter c in the terminal — it copies the login link to your clipboard. Paste it into your browser manually.

4

Your first session

Navigate to your project and start talking

Claude Code works from inside your project folder. Before you type claude, you tell the terminal which folder to look at using the cd command.

What is "cd"?

cd stands for "change directory" — directory is just another word for folder. Typing cd ~/Documents/my-project is the terminal equivalent of double-clicking that folder in Finder.

cmux
# Navigate to your project (update this path to match yours)
cd ~/Documents/my-project

# Start Claude Code
claude
💡
Easy way to get the right path

In Finder, right-click your project folder and hold Option — you'll see "Copy [folder] as Pathname." Paste it after cd .

Your first three moves inside Claude Code

  • 1 Get oriented: Type "What does this project do?" and press Enter. Claude will read your files and explain what it's looking at.
  • 2 Set up your cheat sheet: Type /init and press Enter. Claude scans your project and creates a permanent notes file so it can help you better in every future session.
  • 3 Ask for real help: Describe something you want to build or fix. "Add a contact form to my homepage" or "fix the error on the login page" — plain English is fine.
5

Letting Claude work on its own

How much autonomy you give Claude is up to you

By default, Claude asks for your approval before making any change to your files. You can see exactly what it's about to do before it does it. But if you trust the task, you can give it full freedom.

🙋
Ask first
Claude explains each action and waits for you to approve it. Default. Best when you're learning or doing something new.
📋
Plan first
Claude shows you the full plan, you approve it once, then it executes everything without interruption. Press Shift+Tab to switch.
🚀
Full auto
Claude works start-to-finish without any prompts. Press Shift+Tab twice to switch into this mode.
💡
Start with "Ask first" while you're learning

Reading Claude's explanations before it acts is a great way to understand what it's doing and build trust in its judgment. Once you're comfortable, switch to Plan or Full auto for faster results.

6

Slash commands

Special controls you type inside Claude Code

Any message that starts with / is a command — not something Claude responds to as a question. Think of them as keyboard shortcuts for common actions.

/init
Do this on every new project
Claude scans your project and creates a permanent cheat sheet (CLAUDE.md) so it understands your code better going forward.
/clear
Fresh start, keep your files
Wipes the chat history. Your file changes stay. Use this when Claude seems confused or you want a clean slate.
/compact
Free up "thinking space"
Like /clear, but smarter. Claude summarizes the conversation instead of erasing it — keeps the key context, drops the noise.
/model
Switch Claude versions
Opus is the most capable. Haiku is the fastest and cheapest. Switch mid-session based on the task.
/memory
See what Claude remembers
Shows all the instructions Claude loaded at the start of this session, including your CLAUDE.md files.
/cost
Check your usage
Shows tokens used and estimated cost. Useful if you're on a usage-based plan and want to keep tabs on spend.
/help
See everything available
Full list of every command available. A good thing to run on your first session.
/review
Get a code review
Have Claude review a set of changes and give feedback — like a second pair of eyes on your work.
7

CLAUDE.md — your instruction manual

How you give Claude permanent preferences and rules

Every time you start Claude Code, it looks for a file called CLAUDE.md in your project folder. If it finds one, it reads it before doing anything else. This is how you tell Claude things that should always be true — like "never delete files without asking" or "we use TypeScript, not JavaScript."

What's a .md file?

Markdown (.md) is a simple text file where # makes a heading and - makes a bullet point. You don't need to know the format — /init creates the file for you, and you can edit it like a plain text document in any app.

📄 CLAUDE.md — example
→
# My Project
Title — Claude reads this to orient itself
→
## How to run it
- Start: `npm run dev`
- Tests: `npm test`
Claude uses these commands automatically when it needs to run or test your project
→
## Style rules
- Use TypeScript
- 2-space indentation
Claude follows these every time it writes code, without you having to repeat them
→
## Never do
- Delete files without asking
- Push directly to main
Hard rules — Claude treats these as non-negotiable

Where to put it

LocationWhat it's for
Your project folder ./CLAUDE.mdRules for this project (shared with teammates if you use git)
Home folder ~/.claude/CLAUDE.mdPersonal preferences that apply to every project you work on
./CLAUDE.local.mdPersonal rules for this project only — not shared with others
💡
Don't write it from scratch

Type /init inside a Claude Code session and it auto-generates one from your project. Then open the file and add your own rules on top.

Your first week with Claude Code

A simple progression to build the habit and get real results fast.

Day
1
Get set up and explore

Install everything, run /init on one project, and ask Claude to explain what the project does. No pressure to build anything — just get comfortable with the back-and-forth.

Day
2
Fix one real thing

Find a bug you've been living with, or something on your list that's been too annoying to tackle. Describe it to Claude and let it fix it. Seeing it work on a real problem builds trust fast.

Day
3
Write your CLAUDE.md

Now that you've had a session or two, you'll know what you want Claude to always do (or never do). Run /init if you haven't, then open CLAUDE.md and add a few personal rules.

Day
5
Try an automation

Pick something you do manually every week — generating a report, sending a summary email, updating a spreadsheet — and ask Claude to automate it. This is where the leverage starts to compound.

Day
7
Give it a real project

Hand Claude something ambitious. A new feature, a full page, a multi-step workflow. Give it a real goal and see how far it can go end-to-end.

What Claude Code can and can't do

Setting honest expectations saves frustration later.

✓  It can
  • Read and write any file in your project folder
  • Run terminal commands on your behalf
  • Search and understand your entire codebase
  • Build multi-step automations that run on a schedule
  • Call external APIs (Slack, HubSpot, OpenAI, etc.)
  • Remember your preferences via CLAUDE.md and auto-memory
  • Switch between Anthropic models mid-session
  • Work in multiple languages: Python, JavaScript, SQL, and more
✗  It can't
  • Browse the web in real-time (it doesn't have live internet access by default)
  • See your screen or interact with your mouse and keyboard
  • Run on a phone or tablet — terminal only
  • Work on files outside the folder you launched it from
  • Guarantee perfect code on the first try — review and iterate
  • Replace your judgment on business decisions or ethics
  • Remember anything between sessions without CLAUDE.md or auto-memory

Tips & common hiccups

  • 🧭 Always navigate to your project first. Claude Code only sees files in the folder you started it from. Type cd your-folder before running claude, or it won't find your project.
  • 🔄 Claude seems off-track? Type /clear. It resets the conversation but keeps all your file changes exactly as they are.
  • 🎯 Be specific. "Fix the bug" is vague. "The submit button on the contact form throws an error when the email field is empty — fix it" gives Claude everything it needs to act precisely.
  • 🧠 Want Claude to remember something permanently? Say "remember this across sessions" and it saves it automatically. You can also add rules directly to your CLAUDE.md file.
  • ⏳ Long sessions can feel slow. Type /compact — it summarizes the conversation history to free up space, without losing the thread of what you're working on.
  • 🛑 Claude about to do something wrong? Press Ctrl + C immediately. Changes aren't finalized until Claude writes to a file, and you can always undo with git.
  • 🚀 Starting a brand-new project? Run /init as your very first move. Claude learns your project structure and creates a CLAUDE.md — making every future session smarter from the start.
