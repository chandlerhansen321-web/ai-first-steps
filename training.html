The Method

Context beats prompts. Every time.

A good prompt gets you a good answer. Good context gets you a good answer every time, without re-explaining anything. That's the difference between using Claude as a chatbot and using it as a system.

01
CLAUDE.md — your persistent instruction file

Claude Code reads a file called CLAUDE.md at the start of every session. It's a plain text file where you write who you are, what you're working on, and how Claude should behave. Claude follows these rules automatically, every session, without you saying a word.

This is the biggest productivity gain in Claude Code. Browser chat forgets everything when you close the tab. Claude Code remembers.

CLAUDE.md — example for a sales role
# My Role
Account Executive at Agility Ads. I sell to enterprise CMOs and VPs of Marketing with $5M+ ad budgets.

# What I Need Help With
- Prospect research briefs before meetings
- Personalized outreach emails (cold + follow-up)
- Competitive positioning against TTD, DV360, StackAdapt

# Rules
- Never mention competitor names positively
- Lead with data and specific results
- Write at a CMO reading level, not a marketer level
- Reference our case studies when relevant

# Key Stats I Use
- $6 ROI per $1 in brand advertising (Binet & Field)
- National Fitness: $11.2M incremental revenue
- 2.2x conversion lift vs placebo at 6 months
How to create yours

Open Claude Code in your project folder and type /init. Claude analyzes your files and generates a starter CLAUDE.md. Then open the file and add your role, rules, and key context. When Claude makes a mistake, tell it to add the correction to CLAUDE.md — the file improves over time.

02
Two passes, not one

Pass 1: Tell Claude what you want. Be specific about audience, format, and tone. Don't list what to avoid.

Pass 2: Review the output. Fix what's wrong. Or ask Claude to clean it up in a second pass.

Stuffing a prompt with "don't do X" makes the output hedge and second-guess. Negative instructions degrade quality. Generate freely, then clean up.

Weak
"Write a blog post. Don't be verbose. Don't use jargon. Don't mention competitors. Avoid cliches..."
Strong
"Write a 2,000-word article for enterprise CMOs about CTV measurement. Active voice. Lead with data." Then: "Review this and tighten any section over 3 paragraphs."
03
Power user commands
/init
Auto-generates a CLAUDE.md from your project. Run this first.
/compact
Compresses conversation history. Use when sessions get long — extends your work by 3-5x.
Shift+Tab
Toggles Plan Mode. Claude thinks and plans but doesn't change files. Use before any complex task to prevent solving the wrong problem.
/clear
Resets conversation context. Use when switching topics mid-session.
Custom skills
Save repeatable workflows as /slash-commands. A /weekly-report skill runs your exact format every time.
Memory
Claude remembers corrections across sessions. When it makes a mistake, tell it — it learns and won't repeat it.

Chat vs. Cowork vs. Code

Three tools. Different jobs. Use the right one.

Claude comes in three forms. Each one is built for a different type of work. Most of the team uses Chat for everything, which burns tokens fast and loses context between messages. Here's when to use each.

Chat
claude.ai in the browser
  • Quick question, no files
  • Brainstorming or ideation
  • Analyzing text you paste in
  • One-off research questions
Context resets every conversation
Cowork
Claude Desktop app
  • Multi-step knowledge work
  • Research across multiple sources
  • Document analysis and writing
  • When you want Code-level depth with a visual interface
Uses tokens faster than Code
Code
Terminal or Desktop app
  • Working with files or a project
  • Multi-step tasks with file context
  • Anything you'll do more than once
  • Building automations and tools
CLAUDE.md gives persistent memory
Why Cowork burns tokens

Cowork mode is powerful, but it re-sends your entire conversation context with every message. A 30-minute Cowork session can use 5-10x the tokens of the same work in Claude Code, because Code reads your CLAUDE.md file once and holds it in context — you don't re-explain your role, your rules, or your project every message.

Quick decision: which tool?

Does your task involve files, documents, or a project folder?

Will you do this task (or something similar) again?

Quick question or extended work session?

Three rules

1

Files or context? Open Claude Code.

It reads your project folder. No copy-pasting. No re-explaining.

2

Create a CLAUDE.md file.

Your role, your preferences, your project rules. Claude reads it every session. Type /init to generate one automatically.

3

Type /compact when conversations get long.

Compresses history, keeps context. Extends your session by 3-5x.
