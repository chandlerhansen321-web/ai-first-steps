Your Role
1

Chat, Cowork, and Code

Three tools. Different jobs. Pick the right one.

You already use Claude in the browser. That's Chat. But Claude comes in three forms, and each one is built for different work. Most of the team uses Chat for everything — that's expensive and slow for the kind of work you do.

Chat
claude.ai in the browser
  • Quick question, no files
  • Brainstorming or ideation
  • Analyzing text you paste in
"Write me a quick objection response for a prospect who says they already use TTD."
"What are the top 3 QBR talking points for a fitness brand with declining CTV ROI?"
"Brainstorm 5 LinkedIn hook angles about incrementality testing."
Context resets every conversation
Cowork
Claude Desktop app
  • Multi-step knowledge work
  • Research across multiple sources
  • Deep document analysis
"Research this prospect's company, pull their SEC filings, and draft a pitch angle."
"Analyze these 3 campaign reports and find the common thread."
"Read our last 10 blog posts and identify gaps in topic coverage."
Powerful for one-off deep work
Code
Terminal or Desktop app
  • Working with files or a project
  • Anything you'll do more than once
  • Building tools and automations
"Build a prospect research tool I run before every meeting — it pulls company data and drafts talking points."
"Build a QBR generator that takes client data and outputs formatted slides every quarter."
"Build a content calendar generator that maps posts to our 4 pillars and exports to .xlsx."
CLAUDE.md gives persistent memory
Why Code wins for repeatable work

Chat forgets everything when you close the tab. Cowork is powerful but uses tokens fast on extended sessions. Code reads a CLAUDE.md file that stores your role, rules, and preferences — every session starts with full context. Build something once, reuse it forever.

Quick decision: which tool?

Does your task involve files, documents, or a project folder?

Will you do this task (or something similar) again?

The Method

Three things that make AI output actually good.

Good context beats a good prompt. Every time. These three techniques are the difference between using Claude as a chatbot and using it as a system.

01
CLAUDE.md — your persistent instruction file

Claude Code reads a file called CLAUDE.md at the start of every session. Write who you are, what you're working on, and how Claude should behave. Claude follows these rules automatically, every session, without you saying a word.

CLAUDE.md — example for sales
# My Role
Account Executive at Agility Ads. I sell to enterprise CMOs and VPs of Marketing with $5M+ ad budgets.

# What I Need Help With
- Prospect research briefs before meetings
- Personalized outreach emails (cold + follow-up)
- Competitive positioning against TTD, DV360, StackAdapt

# Key Stats I Use
- $6 ROI per $1 in brand advertising (Binet & Field)
- National Fitness: $11.2M incremental revenue
CLAUDE.md — example for strategy
# My Role
Precision Strategy Consultant at Agility Ads. I manage client relationships and campaign strategy for enterprise accounts.

# What I Need Help With
- Client meeting prep docs
- QBR section drafts
- Campaign performance summaries for client-facing delivery

# Rules
- Write for C-suite audience. Numbers first, plain language.
- Reference specific client results, not generic claims.
CLAUDE.md — example for marketing
# My Role
Marketing at Agility Ads. I manage the blog, social, email, and newsletter content across all channels.

# What I Need Help With
- Blog briefs and social content calendars
- Email nurture sequences for HubSpot
- Brand voice compliance checks

# Voice Rules
- Active voice, human subjects. No em dashes.
- Never mention competitors positively.
- Data-driven, not jargon-heavy.
How to create yours

Type /init in Claude Code. It generates a starter file. Then add your role, rules, and key context. When Claude makes a mistake, tell it to update CLAUDE.md — the file improves over time.

Power move: add examples to CLAUDE.md

Include 1-2 examples of good output in your CLAUDE.md. "Here's what a good prospect brief looks like: [example]." Claude mimics the format every time. This is called few-shot prompting — showing what you want is faster than describing it.

Use @ to point at files

Instead of describing where data is, reference it directly: "Read @proposals/q1-deck.pptx and summarize the key metrics." Or: "Compare @data/q1.csv with @data/q2.csv." Claude reads the files directly — faster and more accurate than pasting content.

03
Two passes, not one

Pass 1: Tell Claude what you want. Be specific about audience, format, and tone. Don't list what to avoid.

Pass 2: Review the output. Fix what's wrong. Or ask Claude to clean it up in a second pass.

Stuffing a prompt with "don't do X" degrades quality. Generate freely, then clean up.

Weak
"Write a cold email. Don't be salesy. Don't use jargon. Don't be too long. Don't mention competitors..."
Strong
"Write a cold email for a VP Marketing at a QSR chain. Reference their recent CTV spend. 3 sentences. Include the $11.2M stat." Then: "Make it more casual — like a peer, not a vendor."
Weak
"Write a client summary. Don't use jargon. Don't be too long. Make it professional..."
Strong
"Write a 1-page Q1 summary for National Fitness. CMO audience. Lead with the 2.2x conversion lift. 5 bullets max." Then: "Replace any adtech terms with plain language."
Weak
"Write a blog post. Don't be verbose. Don't use jargon. Don't mention competitors. Avoid cliches..."
Strong
"Write a 2,000-word article for CMOs about CTV measurement. Active voice. Lead with data." Then: "Tighten any section over 3 paragraphs."
Add a third pass: self-verification

After Claude builds something, ask: "Write a verification checklist for what you just built." Claude generates a list of things to test. This catches edge cases you wouldn't think to check and gives you confidence the output is right before you share it.

Power user commands

/init
Auto-generates a CLAUDE.md from your project. Run this first.
/compact
Compresses conversation history. Add instructions: /compact Focus on key decisions
Shift+Tab
Plan Mode. Claude plans but doesn't change files. Reduces trial-and-error waste by 20-30%.
/clear
Resets context. Use between unrelated tasks.
/btw
Side question that never enters history. Zero context cost.
claude --continue
Resume your last session with full context.
Custom skills
Save workflows as /slash-commands. A /prospect-brief skill runs your exact sequence every time.
75% rule
Sessions under 75% context produce better output. Run /compact at 50%, not when maxed.
claude -p "..."
Headless mode. Runs a one-liner with no interactive session. Perfect for scripting: claude -p "read prospects.csv and generate a brief for each company"
/rc
Remote Control. Continue your terminal session from your phone, tablet, or any browser. Code stays local.
/loop 5m /task
Run a task on a recurring interval. Monitor deploys, poll for new data, babysit a long process.
/effort high
Control thinking depth: low (quick formatting), medium (default), high (complex analysis). Saves tokens on simple tasks.
/context
Shows where your tokens are going — which files, tools, and MCP servers consume the most. Find and fix token drains.
.claudeignore
File that excludes folders from context (like node_modules). Stops Claude from reading irrelevant files. Put one in every project.
Hooks
Automate before/after actions. "After Claude generates a .docx, open it in Word." Configure via /update-config.
/statusline
Custom terminal HUD showing model, context %, git status, and session cost. Always know where you stand.

Picking the right model

H
Haiku — Fast and cheap

Simple formatting, data extraction, quick lookups. Use for high-volume, low-complexity tasks.

S
Sonnet — The default

Most tasks. Good balance of speed and quality. Use /model to switch if you need something else.

O
Opus — Deep reasoning

Complex analysis, long documents, nuanced writing. When quality matters more than speed.

Five things that hurt your output

1
The kitchen sink session

Mixing unrelated tasks in one conversation. Fix: /clear between topics.

2
The correction spiral

3+ corrections on the same issue. Fix: /clear and rewrite the prompt.

3
Vague requests

"Make this better" costs 3 rounds. "Rewrite the intro with the 47% stat, under 80 words" costs 1.

4
Overstuffed CLAUDE.md

~150-200 instruction budget. If it's too long, rules get lost. Cut anything that doesn't prevent mistakes.

5
Skipping Plan Mode

Jumping to execution causes 20-30% more token waste. Press Shift+Tab first.

Four non-developer mistakes

1
Treating Claude like Google

Asking questions instead of giving tasks.

"What's the best way to write cold emails?" → "Write 3 cold email variants for [prospect] using our CLAUDE.md positioning."

"How do I prep for a QBR?" → "Read @national-fitness-q1.csv and draft 3 QBR slides with metrics, wins, and next-quarter recommendations."

"What makes a good LinkedIn post?" → "Write 4 LinkedIn posts about incrementality testing. Hook under 15 words. Data-driven. Export as .xlsx."

2
Describing files instead of pointing to them

"I have a CRM export with deal stages..." → "Read @pipeline_export.csv and find stalled deals over 30 days."

"I have a spreadsheet with usage data..." → "Read @account_usage.csv and flag accounts with declining engagement."

"I have a spreadsheet with campaign data..." → "Read @q1-campaigns.csv and rank channels by CPA improvement."

3
Monolithic prompts

500+ words in one block produces inconsistent results 60% of the time. Break complex tasks into steps: "First do X. Then do Y. Finally do Z."

4
Starting fresh every time

Opening a new session for each task instead of building a project folder. Use claude --continue to resume. Keep templates, data, and CLAUDE.md in one folder.

What Claude Code can and can't do

Can
  • Read and write files on your computer
  • Run terminal commands
  • Search your entire project folder
  • Build automations and scripts
  • Call APIs (HubSpot, Slack, Google, etc.)
  • Remember preferences via CLAUDE.md
  • Generate .docx, .xlsx, .html, .csv, JSON
Can't
  • Browse the web in real-time
  • See your screen
  • Run on your phone
  • Access files outside your project folder
  • Guarantee perfect output every time
  • Remember anything without CLAUDE.md
Integrations (MCP servers)

Claude Code connects to Slack, HubSpot, Google Calendar, Ahrefs, and more via integrations called MCP servers. Ask your admin to set up the ones relevant to your role.
