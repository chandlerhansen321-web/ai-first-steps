Chat, Cowork, and Code

Three tools. Different jobs. Pick the right one.

You don't write code. You describe what you want in plain English — like briefing an extremely fast, extremely thorough intern. Claude does the technical work. The only new thing you learn is where to type.

You already use Claude in the browser. That's Chat. But Claude comes in three forms, and each one is built for different work. Most of the team uses Chat for everything — that's expensive and slow for the kind of work you do.

Chat
claude.ai in the browser
  • Quick question, no files
  • Brainstorming or ideation
  • Analyzing text you paste in
"Write me a quick objection response for a prospect who says they already use TTD."
"What are the top 3 QBR talking points for a fitness brand with declining CTV ROI?"
"Brainstorm 5 LinkedIn hook angles about incrementality testing."
Context resets every conversation
Cowork
Claude Desktop app
  • Multi-step knowledge work
  • Research across multiple sources
  • Deep document analysis
"Research this prospect's company, pull their SEC filings, and draft a pitch angle."
"Analyze these 3 campaign reports and find the common thread."
"Read our last 10 blog posts and identify gaps in topic coverage."
Powerful for one-off deep work
Code
Terminal or Desktop app
  • Working with files or a project
  • Anything you'll do more than once
  • Building tools and automations
"Build a prospect research tool I run before every meeting — it pulls company data and drafts talking points."
"Build a QBR generator that takes client data and outputs formatted slides every quarter."
"Build a content calendar generator that maps posts to our 4 pillars and exports to .xlsx."
CLAUDE.md gives persistent memory
Why Code wins for repeatable work

Chat forgets everything when you close the tab. Cowork is powerful but uses tokens fast on extended sessions. Code reads a CLAUDE.md file that stores your role, rules, and preferences — every session starts with full context. Build something once, reuse it forever.

Claude will ask before acting

By default, Claude Code asks permission before reading files, running commands, or making changes. You'll see a prompt — press Enter or type y to approve. You can also press a to approve all similar actions for the session. This keeps you in control while you learn.

What if something goes wrong?

Press Ctrl+C at any time to stop Claude mid-action. Claude asks permission before editing files, so you can always say no. If a file gets changed incorrectly, Claude can undo its own edits — just tell it "undo that last change." You won't break anything by experimenting.

The Method

Three things that make AI output actually good.

Good context beats a good prompt. Every time. These three techniques are the difference between using Claude as a chatbot and using it as a system.

01
CLAUDE.md — your persistent instruction file

Claude Code reads a file called CLAUDE.md at the start of every session. Write who you are, what you're working on, and how Claude should behave. Claude follows these rules automatically, every session, without you saying a word.

CLAUDE.md — example for sales
# My Role
Account Executive at Agility Ads. I sell to enterprise CMOs and VPs of Marketing with $5M+ ad budgets.

# What I Need Help With
- Prospect research briefs before meetings
- Personalized outreach emails (cold + follow-up)
- Competitive positioning against TTD, DV360, StackAdapt

# Key Stats I Use
- $6 ROI per $1 in brand advertising (Binet & Field)
- National Fitness: $11.2M incremental revenue
CLAUDE.md — example for strategy
# My Role
Precision Strategy Consultant at Agility Ads. I manage client relationships and campaign strategy for enterprise accounts.

# What I Need Help With
- Client meeting prep docs
- QBR section drafts
- Campaign performance summaries for client-facing delivery

# Rules
- Write for C-suite audience. Numbers first, plain language.
- Reference specific client results, not generic claims.
CLAUDE.md — example for marketing
# My Role
Marketing at Agility Ads. I manage the blog, social, email, and newsletter content across all channels.

# What I Need Help With
- Blog briefs and social content calendars
- Email nurture sequences for HubSpot
- Brand voice compliance checks

# Voice Rules
- Active voice, human subjects. No em dashes.
- Never mention competitors positively.
- Data-driven, not jargon-heavy.
How to create yours

Type /init in Claude Code. It generates a starter file. Then add your role, rules, and key context. When Claude makes a mistake, tell it to update CLAUDE.md — the file improves over time.

Power move: add examples to CLAUDE.md

Include 1-2 examples of good output in your CLAUDE.md. "Here's what a good prospect brief looks like: [example]." Claude mimics the format every time. This is called few-shot prompting — showing what you want is faster than describing it.

Use @ to point at files

Instead of describing where data is, reference it directly: "Read @proposals/q1-deck.pptx and summarize the key metrics." Or: "Compare @data/q1.csv with @data/q2.csv." Claude reads the files directly — faster and more accurate than pasting content. Just type @ followed by the filename — Claude auto-completes it for you.

03
Two passes, not one

Pass 1: Tell Claude what you want. Be specific about audience, format, and tone. Don't list what to avoid.

Pass 2: Review the output. Fix what's wrong. Or ask Claude to clean it up in a second pass.

Stuffing a prompt with "don't do X" degrades quality. Generate freely, then clean up.

Weak
"Write a cold email. Don't be salesy. Don't use jargon. Don't be too long. Don't mention competitors..."
Strong
"Write a cold email for a VP Marketing at a QSR chain. Reference their recent CTV spend. 3 sentences. Include the $11.2M stat." Then: "Make it more casual — like a peer, not a vendor."
Weak
"Write a client summary. Don't use jargon. Don't be too long. Make it professional..."
Strong
"Write a 1-page Q1 summary for National Fitness. CMO audience. Lead with the 2.2x conversion lift. 5 bullets max." Then: "Replace any adtech terms with plain language."
Weak
"Write a blog post. Don't be verbose. Don't use jargon. Don't mention competitors. Avoid cliches..."
Strong
"Write a 2,000-word article for CMOs about CTV measurement. Active voice. Lead with data." Then: "Tighten any section over 3 paragraphs."
Add a third pass: self-verification

After Claude builds something, ask: "Write a verification checklist for what you just built." Claude generates a list of things to test. This catches edge cases you wouldn't think to check and gives you confidence the output is right before you share it.

Under the Hood

How the technology works. Read this when you're curious.

Now that you've used the tools, it helps to understand what's actually happening when you type a prompt. This isn't computer science class — just the mental model that makes everything click.

The prediction engine

An LLM (Large Language Model) is a text prediction engine. It was trained on most of the internet and learned patterns in language — how sentences flow, how arguments are structured, how code works. When you type a prompt, it predicts the most useful response token by token. Think of it as autocomplete that went to graduate school.

Tokens and the context window

Claude thinks in tokens, not words. A token is roughly 3/4 of a word. Claude's working memory — the context window — holds about 1 million tokens. That's roughly 3,000 pages of text. Everything in your conversation plus every file Claude reads takes up space in that window. When it fills up, Claude summarizes earlier messages to make room.

Chatbot vs. agent

ChatGPT and Claude Chat are chatbots — you ask a question, they answer. Claude Code is an agent. The difference: an agent doesn't just talk. It takes action.

Read
Understand your request and read files
Plan
Decide what to do next
Act
Run a command or write a file
Check
Verify it worked, then repeat

This loop — read, plan, act, check — runs continuously until the job is done. Claude might read 10 files, run 5 commands, and edit 3 documents before responding. That's what makes it useful for real work, not just answering questions.

Ask vs. Build

Most people use AI to ask questions and get answers. That's useful, but it's the smallest thing Claude Code can do. The real power is building things that keep working after you close the tab.

Ask
"Analyze this CSV and tell me which deals are stalled."
You get an answer. Next week, you ask again. And again.
Build
"Build me a script that reads my pipeline export, flags stalled deals, and emails me a summary every Monday."
You build it once. It runs forever. You never ask again.

Scripts, tools, dashboards, quality checks, automated reports — these are real files on your computer that you run whenever you need them. That's the difference between using AI as a search engine and using it as a builder.

What Claude Code can do

Claude Code uses tools to take real action — the same way you use apps on your phone. Here's what it can work with:

Run commands
Install software, process data, start servers, run your scripts — anything you'd type in a terminal.
Read files
Opens and understands any file in your project — documents, spreadsheets, PDFs, images, data files.
Create files
Writes new files from scratch — scripts, reports, web pages, spreadsheets, presentations.
Edit files
Makes precise changes to existing files without rewriting the whole thing.
Search your project
Finds files by name or searches inside files for specific text — across hundreds of files instantly.
Search the web
Pulls current information from the internet — company news, market data, competitor updates.

Your files, your computer

Claude Code works inside a folder on your computer. It reads files, creates new ones, and saves everything locally. When Claude "builds" something — a script, a report, a web page — it's writing actual files to your hard drive. Not generating text that disappears when you close the tab. That's why you navigate to a project folder before starting Claude Code: that folder is where everything lives.

How your data is handled

Files Claude reads are sent securely to Anthropic's servers for processing over encrypted connections, then discarded. Files Claude doesn't access stay entirely on your machine. Code execution and file writing happen locally. On Pro, Max, Team, and Enterprise plans, your data is never used to train AI models. For sensitive client data, check with your admin about your organization's data policies.
